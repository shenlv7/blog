## 四、Float Drone 浮空飞行器 ⚠️ 已修正

### ⚠️ 原方案问题与修正

> **原方案声称**：550g机体，氦气浮力抵消82%自重，悬停功耗仅3W

#### 问题1：氦气体积严重低估

**原方案计算**：450g浮力 → 需要多少氦气？

```
氦气净升力 ≈ 1.06 kg/m³（标准大气压，20°C）
450g ÷ 1.06 kg/m³ ≈ 0.425 m³ = 425升
球体直径 = 2 × ∛(3V/4π) ≈ 93cm
```

**❌ 问题**：直径93cm的球体在室内（层高2.7m）几乎无法自由移动，门框都过不去。

#### 问题2：充电续航计算矛盾

**原方案声称**：充电15分钟 → 续航2小时

```
充电功率 15W × 0.25h = 3.75 Wh
巡航功耗 8W → 续航 = 3.75 ÷ 8 = 0.47h ≈ 28分钟 ❌
悬停功耗 3W → 续航 = 3.75 ÷ 3 = 1.25h ≈ 75分钟 ❌
```

**结论**：原方案续航计算有误，实际续航远低于声称值。

#### 问题3：机械臂反作用力

气球上安装机械臂伸出去推东西，根据牛顿第三定律，气球会被反推。100g夹持力 = 0.98N反作用力，虽然不大，但会导致平台漂移，需要更强的悬停控制。

---

### ✅ 修正方案：混合升力架构

**核心思路**：小氦气球（提供部分升力）+ 主动推力（补偿剩余重量）

#### 修正后参数

| 参数 | 原方案（❌） | 修正方案（✅） | 变化 |
|------|------------|-------------|------|
| 总重量 | 550g | 680g | +130g（更真实的组件重量） |
| 氦气浮力 | 450g (82%) | 200g (29%) | 大幅减少 |
| 球体直径 | ~93cm | **~72cm** | 可过门框 |
| 悬停功耗 | 3W | **8W** | 更现实 |
| 巡航功耗 | 8W | **15W** | 含传感器全负载 |
| 电池容量 | 未说明 | **40Wh (4S1P 18650)** | 明确规格 |
| 悬停续航 | 12h | **4-5h** | 实际可达 |
| 巡航续航 | 4-6h | **2-2.5h** | 实际可达 |
| 充电15分钟续航 | 2h ❌ | **30-40分钟** ✅ | 修正计算 |

#### 升力分配

```
总重 680g
├── 氦气浮力: 200g (29%) ← 72cm球体，约190升
├── 4×电机推力: 480g (71%) ← 每电机120g推力
└── 悬停实际需求: ~550g ← 有130g推力裕度
```

## 五、完整工作流

### 核心闭环

```
01  Float Drone 全屋巡航 · 采集多模态数据
         ↓
02  感知数据加密本地内网 → Home Core Hub
    （WiFi6 私有网络 · AES-256 传输加密）
         ↓
03  多模态模型实时融合理解
    （场景识别 | 用户意图 | 情绪状态 | 异常检测）
         ↓
04  决策引擎生成执行指令
    ① 飞行器：投影/语音/跟随
    ② 全屋家电：灯光/空调/窗帘
    ③ 桌面：OLED同步
         ↓
05  飞行器持续反馈现场状态 · 算力中心动态修正
    （闭环刷新频率 2Hz · 延迟＜500ms）
```
