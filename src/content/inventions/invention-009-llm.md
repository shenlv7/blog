# 🔧 发明 009：大语言模型(LLM)实战笔记

## 背景

在学习LLM理论之后，我决定动手实践——从本地运行开源模型到API调用，从理解采样策略到优化推理性能。

## 实验过程

### 第一步：本地运行开源模型

使用ollama运行Qwen2.5-7B：

```bash
ollama run qwen2.5:7b
```

惊喜发现：7B参数的模型在普通笔记本上就能流畅对话！虽然推理速度不如API快，但完全可用。

### 第二步：理解采样策略

通过对比实验，我发现不同参数对输出质量影响巨大：

- **temperature=0.1**：输出确定、单调，适合事实性任务
- **temperature=0.7**：平衡创造性与连贯性，通用推荐
- **temperature=1.0+**：创意写作，但容易跑偏

top_p和top_k的配合也很关键，核采样能有效避免低概率token的干扰。

### 第四步：量化实验

## 关键发现

1. **MoE架构是未来**：DeepSeek-V3的671B参数中，每次只激活37B，既保证能力又控制成本
2. **上下文窗口不是越长越好**：超过一定长度后，模型的"中间遗忘"问题很明显
3. **小模型+好Prompt > 大模型+烂Prompt**：提示工程的威力被低估了

## 遇到的坑

- 显存不足：7B模型需要约14GB显存（FP16），量化后可以降到4GB
- 模型加载慢：首次加载需要时间，适合用API服务而非每次重新加载
- 中文支持：部分英文模型的中文能力有限，推荐使用Qwen、ChatGLM等中文优化模型

## 下一步计划

2. 实验RAG（检索增强生成）系统
3. 探索Agent框架（LangChain、AutoGen）

## 收获

通过这次实践，我真正理解了LLM的"炼金术"——它不是魔法，而是工程+规模+数据的胜利。掌握这些底层原理，才能更好地使用和优化大模型。
