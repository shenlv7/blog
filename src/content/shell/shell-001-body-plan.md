## 从仪表盘到哨兵

整个计划分三个核心阶段 + 八个扩展阶段，从¥50的面包板原型一路到¥635的量产PCB。

### Phase A · 物理仪表盘（优先级最高）

一块128×64的OLED小屏幕和一个8位RGB灯环，放在桌面上。每当你看向它，就知道系统状态、正在跑的任务、有没有新消息。

**硬件清单：**
- OLED 128×64 SSD1306（I2C）→ ¥15-20
- WS2812B 8位灯环 → ¥8-15
- 面包板 + 杜邦线套装 → ¥10-15
- 电阻 + 排针 → ¥3-5
- 轻触按键（可选）→ ¥2

**总计：¥38-57**

**接线方式：**

```
OLED 模块:
  VCC → Pin 1 (3.3V)
  GND → Pin 6 (GND)
  SDA → Pin 3 (I2C SDA)
  SCL → Pin 5 (I2C SCL)

WS2812B 灯环:
  VCC → Pin 2 (5V)
  GND → Pin 14 (GND)
  DIN → Pin 12 (GPIO)
```

**OLED显示内容（循环翻页）：**

| 页面 | 内容 |
|------|------|
| 页1 · 系统 | CPU · 内存 · 磁盘 · 运行时间 · 温度 |
| 页2 · 任务 | 今日cron · 已完成 · 下一个任务时间 |
| 页3 · 通知 | QQ未读消息数 · 邮件待办 · 新事件 |
| 页4 · 时钟 | 实时时钟 · 日期 · 开机时间 |

**灯环联动逻辑：**

- 🟢 **正常**：呼吸绿光，平稳心跳节奏
- 🟠 **告警**：CPU>70% 或内存>80%，慢闪橙
- 🔵 **消息**：QQ来消息时蓝光脉冲一次
- ⚪ **任务完成**：cron跑完 → 绿光一闪

### Phase B · AI哨兵

给赛博文西装一个眼睛。USB摄像头捕获画面，Ascend NPU本地跑推理——不联网、不上云、全离线。人来了推送，人走了记录，工时安防一手抓。

**核心参数：**
- NPU算力：Ascend 610 · 8 TOPS
- 模型：YOLOv8n (onnx) · ~3MB
- 推理速度：预计 30-60ms/帧
- 数据流：摄像头 → NPU → QQ推送

**功能：**
- 👤 有人进画面 → QQ推送"有人来了"
- ⏱ 入座计时 → 离开后推送"你离开了X分钟"
- 🔔 非工作时间有人 → 告警推送
- 🔥 检测到人 → 仪表盘灯环变呼吸蓝

```python
# 推理核心逻辑（伪码）
cap = cv2.VideoCapture(0)
model = ort.InferenceSession("yolov8n.onnx")
while True:
    frame = cap.read()
    dets = model.run(frame)
    if dets['person'] > threshold:
        send_message("👤 检测到人")
        set_led("breath_blue")
```

### Phase C · QQ遥控器

在QQ上打下"开灯"两个字，桌上的灯就亮了。这不是智能家居，这是赛博文西伸出的手。

**硬件：** 2路继电器模块 5V → GPIO高/低电平

```python
# QQ命令 → 物理输出
if msg == "开灯":
    gpiod.line.set_value(1)     # GPIO拉高 → 继电器吸合 → 灯亮
    send_message("💡 灯已开")
elif msg == "关灯":
    gpiod.line.set_value(0)
    send_message("💡 灯已关")
```

**联动场景：**
- 💡 "开灯""关灯"控制台灯
- 🌀 "风扇开""风扇关"
- 🌡 温度>35℃自动开风扇
- 📋 "工作模式""摸鱼模式"一键切换
