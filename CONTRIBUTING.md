# 博客贡献指南

## 📝 写作风格

### 语言风格
- 通俗易懂，避免过度学术化
- 使用类比和实际案例解释概念
- 保持专业但亲切的语气
- 适当使用emoji增加可读性

### 文章结构
```markdown
---
title: "文章标题"
description: "简短描述（100-150字）"
pubDate: YYYY-MM-DD
tags: ["标签1", "标签2"]
---

## 引言
简要介绍主题和文章价值

## 核心概念
详细讲解主要内容

## 代码示例
```python
# 实际可运行的代码
```

## 实践建议
如何应用到实际项目

## 总结
要点回顾和下一步学习建议

## 参考资料
- 链接1
- 链接2
```

## 🖼️ 配图规范

### 图片来源
- Unsplash: https://unsplash.com
- Pexels: https://pexels.com
- 必须免费可商用

### 图片格式
```markdown
![描述](https://images.unsplash.com/photo-xxx?w=800&h=400&fit=crop)
```

### 图片位置
- 每篇文章 3-5 张配图
- 关键概念处插入
- 避免大段文字无图

## 🏷️ 标签规范

## 📅 发布流程

1. **生成文章**：每周三自动生成
2. **审核内容**：检查准确性和质量
3. **添加配图**：确保图片链接有效
4. **推送到GitHub**：
   ```bash
   cd blog
   git add src/content/posts/new-article.md
   git commit -m "post: 新文章标题"
   git push
   ```
5. **自动部署**：GitHub Actions 自动构建发布
6. **邮件推送**：发送到订阅者邮箱
